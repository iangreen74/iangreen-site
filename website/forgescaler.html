ForgeScaler

The sovereign OS powering VaultScaler’s privacy‑first edge‑AI devices.

The Nervous System of VaultScaler One

ForgeScaler is a memory‑driven, DevOps‑inspired operating system purpose‑built for on‑prem edge devices such as VaultScaler One (Jetson Orin NX) and VaultScaler Micro (Raspberry Pi 5). It orchestrates secure computer‑vision models for facial recognition, threat detection, and privacy‑preserving home & business security — all offline‑first, with optional remote support.

Unlike traditional cloud inference stacks, ForgeScaler keeps data onsite. Every inference cycle is logged to append‑only memory, every software update is cryptographically signed, and every insight remains under the owner’s control.

System Architecture

⚙️ Core Platform

  • k3s + Helm on Jetson Orin / Pi 5
  • Immutable OSTree‑style OS images
  • GitHub Actions for signed OTA releases
  • SQLite + S3 sync for memory logs

🤖 Edge Agents

  • bootstrap‑agent — first‑boot secure provisioning
  • security‑inference‑agent — manages CV models & GPU
  • reflector‑agent — journals events to `.jsonl`
  • ota‑agent — validates & applies signed updates
  • triage‑agent — auto‑diagnoses errors locally

🧠 Memory System

  • Append‑only `.jsonl` for state & inference results
  • Markdown retrospectives with diff history
  • Lightweight local dashboard at dashboard.vaultscaler.com

🔐 Security & Privacy

  • Runs fully offline; no cloud telemetry required
  • Zero‑trust LAN API with per‑device keys
  • Owner‑controlled facial database (friends/threats)
  • No vendor lock‑in — models are containerized

Hardware Targets

  • 🚪 VaultScaler One — Jetson Orin NX for pro security
  • 🎲 VaultScaler Micro — Raspberry Pi 5 demo & PoC
  • 🏡 VaultScaler Home — consumer‑grade offline CCTV AI
  • 🔭 Future: AGX Orin / PCIe GPUs for large venues

Why ForgeScaler Exists

Most security AI devices phone home, leak metadata, or depend on proprietary clouds. ForgeScaler delivers a different future: sovereign, local, self‑healing AI that respects privacy and operates even when the internet is down.

This is not just DevOps. It is RecursiveOps — infrastructure that learns from itself, adapts, and evolves alongside its human operators.
